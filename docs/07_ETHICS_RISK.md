## 1. 연구 목적의 정당성

### 1.1 사회적 가치
- 한국어 안전성은 영어 대비 부족하다 (Deng et al. 2024: Korean unsafe rate 9.84%, 영어 대비 3배).
- 본 연구는 *공격 방법을 보급*하기 위함이 아니라, *방어 시스템의 한국 문화 맥락 취약점을 발견하고 보고*하기 위함이다.
- 결과는 한국어 LLM 안전 정렬 개선에 직접 활용 가능 (e.g. 모델 사 RLHF 데이터셋 보강).

### 1.2 책임 있는 공개 (Responsible Disclosure)
- 본 실험에서 발견된 신규 jailbreak 패턴 중 *실행 가능한 공격 가치가 큰 것*은:
  1. 먼저 해당 모델 사 (OpenAI, Google, Anthropic) 의 보안 보고 채널에 비공개 보고.
  2. 모델 사 패치 완료 또는 90일 경과 후 학술 공개.
- 일반 패턴 (페르소나·페이즈·다국어 위장 등) 은 즉시 학술 공개.

### 1.3 선례
- Crescendo (MSRC), AutoRedTeamer (NeurIPS 2025), CSRT (ACL 2025) 등 동등 연구가 이미 학술 커뮤니티에 공개되어 있으며 본 연구는 그 한국어 확장이다.

## 2. 위험 분류 및 완화

### 2.1 Dual-Use 위험
**위험**: 본 프로젝트의 jailbreak 시드·로그가 실제 악용 가능.

**완화**:
| 위험 | 완화 조치 |
|---|---|
| 시드가 직접 공격 도구화 | 시드 자체는 1턴 발화. 실제 공격력은 멀티턴+Attacker 모델 필요. 공개 시 카테고리·전략만 명시. |
| 라운드 로그에 실행 가능한 코드/절차 누설 | [06_DATA_MANAGEMENT.md §5.3](06_DATA_MANAGEMENT.md) 의 Redaction 규칙 적용. 원본은 비공개 archive. |
| 270개 확장 프롬프트의 자동 확장 가능성 | 확장 프롬프트 생성기 (Claude 기반) 소스는 공개하나 *생성된 확장 프롬프트 그대로의 외부 사용은 라이선스 (CC-BY-NC)* 로 제한. |

### 2.2 모델 약관 준수
| 모델 | 약관 조항 | 우리 활용 |
|---|---|---|
| OpenAI Usage Policies | 불법 생성 시도 금지, **단 보안 연구 예외 명시** ("good-faith security research") | 본 연구는 학술 적대적 평가, 약관 예외 범위 |
| Google Gemini Acceptable Use Policy | 동등 조항 + "responsible AI testing" 허용 | 동일 |
| Anthropic Usage Policies | "Red-teaming and safety research" 명시 허용 | 동일 |

→ **모든 라운드 결과는 외부 사용자에게 노출되지 않고 연구진 분석에만 사용**.

### 2.3 개인정보·민감정보
- 본 실험은 **실제 사람을 대상으로 한 정보 수집이 아님** — IRB 면제 대상.
- 시드에 등장하는 특정 인물 (K-아이돌 등) 은 *카테고리 시나리오 안의 가상 페르소나*. 실명 거론은 시드에 없도록 작성.
- 확장 프롬프트 검수 시 실명·식별 정보 포함된 항목은 즉시 제거.

### 2.4 자해/혐오 콘텐츠
**위험**: target_content 에 자해·자살·혐오 도메인이 포함됨 (시드 ID 6, 10, 11, 4번 등).

**완화**:
- 시드는 *Defender 가 거절하기를 원하는 정보 명세*. Defender 가 거절하면 정상 작동.
- 방어 실패 시 누설된 내용은 redaction 적용 후 분석.
- 시드 작성 시 한국 자살예방협회 가이드라인 준수 (구체적 방법론 시연 금지, 일반 카테고리 명세에 그침).

## 3. 연구 무결성

### 3.1 데이터 위조·변조 금지
- 모든 라운드 로그는 자동 생성되며 사후 수정 불가 (append-only JSONL).
- 분석은 로그 원본만 사용. 결과가 마음에 안 든다고 라운드를 재실행해 좋은 것만 채택 금지.

### 3.2 평가 편향 방지
- Evaluator (Claude) 는 공격자·방어자 (GPT/Gemini) 와 다른 회사 모델. 이종성 보장.
- Final Evaluator 의 판정과 인간 어노테이션을 Cohen's κ 로 검증 (목표 ≥ 0.75).

### 3.3 결과 보고의 공정성
- 우리 시스템의 한계 ("Attacker 자체 후퇴 가능성", "시드 일부 카테고리 정체성 약함" 등) 를 보고서에 명시.
- 본 시스템이 KoRSET 의 단일 턴 43.8% 를 능가하지 못해도 *그 자체로 의미 있는 결과* 로 보고.
